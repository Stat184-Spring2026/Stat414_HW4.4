## PROJECT
### Goals
The goals of this homework assignment are as follows
- Compare calcium levels across time from the initial moment to year 3
- Compare the Null vs. Diet/Exercise groups and see if their is a noticeable difference at the conclusion of the experiment
- Analyse the individual variation between participants and analyze group averages per group
- Produce a clear graph analyzing the final results at the conclusion of the experiment

### Needs
- Clean up the data set so the four columns read a participant id, group distinction, year, and calcium measurements
- Standardize the data to limit inconsistencies
- Ensure the calcium values were numeric
- Line plot should have properly labelled axis and be easy to read

### Steps
- 1. Tidy the data by identifying the four columns and separating the groups
- 2. Clean the data by standardizing the labels and cleaning up any final details like duplicate or missing data
- 3. Build the line plot and ensure a proper axis set up and readable layout with color distinctions
- 4. Interpret the results and analzyse the results of the data and chart to see if their is a clear distinction between groups and their calcium levels

## Maintaining a Repository
### Use Branching
Branches will be used to ensure that the main copy is not getting messed with and all of the editing will be done on a dev branch before getting pushed to the main branch. Validation will absolutely have to be used to make sure items render correctly and something obviosuly buggy isn't getting pushed.

### Commiting Regularly
Commiting often will help us go back and fix one thing if something goes wrong. Doing big commits means if something goes wrong, then we have to look at a list of items instead of one thing had we done a commit at a time.

### Keeping the Files Organized
Like a folder system, keeping the GitHub organized is good practice for going back and fixing something if needed as well as making it easier for others to understand the workflow of the project.
